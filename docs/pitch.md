## 슬라이드 3 — 라이브 데모 (01:00 ~ 03:30, 2분 30초)

### Step 1 — 메인 페이지 (30초)
- 예산 30,000원 · 받는 사람 **부모님** · 자리 **생일** · 취향 **건강 챙기시는 분**
- "AI 선물 세트 추천받기" 클릭

### Step 2 — AI 추천 결과 (45초)
- 세트 3개가 카드로 표시
  - 세트 #1: "건강을 담은 아침 한 잔" — 아산 인삼차 + 계룡산 꿀
  - 세트 #2: "전통의 향기" — 청양 고추장 + 천안 호두과자
  - 세트 #3: "달콤한 휴식" — 부여 연꽃 한과 + 논산 딸기잼
- 각 세트의 스토리·합계가 한눈에

### Step 3 — 상품 상세 (30초)
- 카드 클릭 → 소상공인 정보 + 상품 설명
- "이 가게는 충남 계룡시 두마면 농원길 12에서 직접 채밀한..."

### Step 4 — 장바구니 + 결제 (30초)
- "세트 담기" → 장바구니
- "주문하기 (데모)" → 결제 완료 화면

## 슬라이드 4 — 보안 시연 (03:30 ~ 04:30, 1분)

> "보안 트랙도 함께 공략했습니다. 실제로 공격을 시도해보겠습니다."

### 라이브 실행 (각 10초)

```bash
./xss.sh        # <script> 삽입 → 정제되어 텍스트만 남음
./csrf.sh       # 토큰 없는 결제 → 403
./bruteforce.sh # 6회 실패 → 429 + 15분 잠금
```

| 공격 | 결과 |
|---|---|
| XSS | DOMPurify 가 `<script>` 제거 |
| CSRF | double-submit cookie 토큰 검증 → 403 |
| SQL Injection | Supabase ORM Prepared Statement 자동 처리 |
| SSRF | URL allowlist + 내부 IP 차단 → 400 |
| BruteForce | 5회 실패 시 15분 잠금 → 429 |
| RBAC | Next middleware + Supabase RLS 이중 방어 |
| 파일 업로드 | MIME + 확장자 + **매직넘버** 3중 검사 |

> "관리자 페이지의 BruteForce 모니터링에서 실시간으로 시도가 기록됩니다."
