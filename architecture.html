시나리오 ② 로그인 POST /api/auth/login

B 이메일/패스워드 → BruteForce 가드 → JWT 쿠키 발급
로그인 시도가 어떻게 차단/통과되고, 세션 쿠키가 어떻게 브라우저에 박히는지.
👤 UserLoginForm
▲ Edgemiddleware
⚡ Route Handler/api/auth/login
🛡️ Securitybruteforce.ts
🗄️ Postgreslogin_attempts
🔐 Supabase Authauth.users
1 POST {email, password} + csrf
2 CSRF 검증 통과 → forward
3 isLockedOut(email) 호출
4 select count where attempted_at > now()-5m
5 ← attempts < 5 → OK
6 supabase.auth.signInWithPassword
7 ← session + JWT
8 insert login_attempts(success=true)
9 ← Set-Cookie: sb-access-token (HttpOnly)
10 ← 200 → router.refresh()
